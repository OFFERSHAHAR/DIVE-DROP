===================================
  GMAIL MCP SERVER - SETUP GUIDE
===================================

This guide walks you through setting up the Gmail MCP server safely
without getting your account blocked.

PREREQUISITES
-------------
- Python 3.10+
- A Google account

STEP 1: Install Dependencies
-----------------------------
Open a terminal in this folder and run:

    pip install -r requirements.txt

STEP 2: Enable Gmail API & Get Credentials
-------------------------------------------
You need to create a Google Cloud project and enable the Gmail API:

1. Go to https://console.cloud.google.com/
2. Click "Create Project" (or select an existing one)
3. Go to APIs & Services > Library
4. Search for "Gmail API" and click Enable
5. Go to APIs & Services > Credentials
6. Click "Create Credentials" > "OAuth client ID"
7. If prompted, configure the consent screen:
   - User type: External (or Internal if using Google Workspace)
   - App name: "Gmail MCP" (or anything you like)
   - Add your email as a test user
8. Application type: "Desktop app"
9. Name: "Gmail MCP"
10. Click "Create"
11. Download the JSON file
12. Rename it to "credentials.json" and place it in this folder

STEP 3: Run OAuth Setup
------------------------
This will open a browser to authenticate your Google account:

    python oauth_setup.py

- Sign in with your Google account
- Click "Advanced" > "Go to [app name] (unsafe)" (since it's unverified)
- Grant read-only access to your Gmail
- Close the browser tab when done

STEP 4: Use the Server
-----------------------
The MCP server is now ready.

To run it with an MCP client (e.g., opencode, Claude, or any MCP host):

    python gmail_mcp.py

Or add it to your MCP client configuration (e.g., opencode.json):

    {
        "mcpServers": {
            "gmail": {
                "command": "python",
                "args": ["path/to/gmail_mcp.py"]
            }
        }
    }


WHY YOU WON'T GET BLOCKED
--------------------------
1. READ-ONLY SCOPE
   We only request gmail.readonly scope. We never send, delete, or
   modify emails. Google is fine with readonly access.

2. OFFICIAL GOOGLE API
   We use google-api-python-client, the official Google library.
   This is the same SDK Google provides for Gmail integrations.
   No scraping, no IMAP hacks, no unofficial methods.

3. RATE LIMITING
   - Exponential backoff on 429 (rate limit) errors
   - Max retries with jitter to avoid thundering herd
   - Results capped at 100 per request
   - Gmail API quota: 1,000,000,000 queries per day
     (you won't hit this from normal use)

4. OAUTH 2.0 BEST PRACTICES
   - Token stored locally in .gmail_token/token.json
   - Automatic token refresh
   - No password storage
   - Minimal permissions (read only)

5. LOCAL ONLY
   - Everything runs on your machine
   - No third-party servers involved
   - Tokens never leave your computer


AVAILABLE TOOLS
---------------
- gmail_list_messages    - List inbox messages with filters
- gmail_get_message      - Get full email content by ID
- gmail_search_messages  - Search using Gmail search syntax
- gmail_list_labels      - List all labels/folders
- gmail_get_unread_count - Quick unread count check
- gmail_list_attachments - List attachments on a message
- gmail_get_profile      - Get account profile info


TROUBLESHOOTING
---------------
"credentials.json not found"
    -> Follow Step 2 above to create and download credentials.

"Access blocked: ..."
    -> You may need to add your email as a test user in the
       Google Cloud Console > APIs & Services > OAuth consent screen.

"Token has expired"
    -> Run "python oauth_setup.py" again to refresh the token.

"Rate limit exceeded"
    -> Wait a minute and try fewer requests. The server already
       handles this with automatic retries.
