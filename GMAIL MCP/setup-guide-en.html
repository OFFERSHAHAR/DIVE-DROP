Gmail MCP Server

Step-by-step setup guide

Read-only • Safe • Official API
0

Check prerequisites

Make sure you have these before starting:

  • Python 3.10+ installed — run python --version to check
  • A Google account (Gmail)
  • All files from the opencode/ folder
Already set up: Dependencies are already installed. Run pip install -r requirements.txt if you move to a new machine.
1

Go to Google Cloud Console

Open the Google Cloud Console to create a project and enable the Gmail API.

https://console.cloud.google.com/

Sign in with the Google account whose emails you want to read.

2

Create a project

At the top of the page, click the project dropdown (it might say "Select a project" or show a project name).

[Project selector dropdown — top bar of Google Cloud Console]
  • Click "New Project"
  • Name it "Gmail MCP" (or anything you like)
  • Click "Create"
  • Wait a few seconds, then select your new project from the dropdown
3

Enable the Gmail API

Navigate to the API Library and enable Gmail API for this project.

// In the left sidebar menu: APIs & Services → Library
  • Search for "Gmail API"
  • Click on the "Gmail API" card
  • Click the blue "Enable" button
Tip: Wait for the "API enabled" notification before moving on.
4

Configure OAuth consent screen

Before creating credentials, set up the consent screen users will see when authenticating.

// In the left sidebar menu: APIs & Services → OAuth consent screen
  • User Type: Select "External" (even if you're the only user)
  • Click "Create"
  • App name: "Gmail MCP"
  • User support email: Your email address
  • Developer contact info: Your email address
  • Click "Save and Continue"
  • Scopes: Skip (click "Save and Continue")
  • Test users: Click "Add Users", add your own Gmail address, then click "Save and Continue"
  • Review and click "Back to Dashboard"
Important: You must add your email as a test user, otherwise authentication will fail with "Access blocked".
5

Create OAuth credentials

Create the "Desktop app" credentials that let your local Python script access Gmail.

// In the left sidebar menu: APIs & Services → Credentials
  • Click "Create Credentials" at the top
  • Select "OAuth client ID"
  • Application type: "Desktop app"
  • Name: "Gmail MCP"
  • Click "Create"

A dialog will pop up with your new Client ID and Client Secret.

[OAuth client created dialog with Download button]
  • Click the "Download JSON" button
  • Click "OK" to close the dialog
6

Place credentials.json in the folder

Take the downloaded JSON file and put it in the right place.

  1. Find the downloaded file (likely named client_secret_....json)
  2. Rename it to credentials.json
  3. Move it into your opencode/ folder on the Desktop
// Your folder should now look like: Desktop/opencode/ ├── credentials.json <-- you just added this ├── gmail_mcp.py ├── oauth_setup.py ├── login.html ├── setup-guide.html ├── setup-guide-en.html ├── setup_guide.txt └── requirements.txt
7

Run OAuth setup

Open a terminal in the opencode/ folder and run the setup script.

C:\Users\GamingPC\Desktop\opencode> python oauth_setup.py

A browser window will open asking you to sign in to your Google account.

[Google sign-in screen → "Choose an account"]
  • Select your Google account
  • Click "Advanced" (the app is unverified — that's expected for personal projects)
  • Click "Go to Gmail MCP (unsafe)"
  • Click "Continue" to grant read-only access
Success! You'll see: "Authentication successful!" in the terminal. A .gmail_token/ folder was created with your saved token.
8

Connect the MCP server

Add the Gmail MCP server to your MCP client config (e.g., opencode.json or claude_desktop_config.json).

{ "mcpServers": { "gmail": { "command": "python", "args": [ "C:\\Users\\GamingPC\\Desktop\\opencode\\gmail_mcp.py" ] } } }

Restart your MCP client. You should see the Gmail tools available:

  • gmail_list_messages
  • gmail_get_message
  • gmail_search_messages
  • gmail_list_labels
  • gmail_get_unread_count
  • gmail_list_attachments
  • gmail_get_profile
9

Try it out

Ask your MCP assistant to use the tools. Example prompts:

  • "Show me my unread emails"
  • "Search for emails from john@example.com"
  • "How many unread messages do I have?"
  • "List my Gmail labels"
  • "Get the full content of message ID xxx"
  • "Show me attachments on the latest email"
Done! You're now reading your Gmail safely through the official API.
!

Why you won't get blocked

  • Read-only scope: We only request gmail.readonly. No sending, deleting, or modifying emails.
  • Official Google API: Using google-api-python-client — the same SDK Google maintains. No scraping or IMAP hacks.
  • Exponential backoff: On rate limit (429) errors, we wait longer with each retry + random jitter.
  • OAuth 2.0: No passwords stored. Token auto-refreshes. Saved locally in .gmail_token/.
  • Local only: Everything runs on your machine. No third-party servers involved.
  • Generous quota: Gmail API allows 1 billion queries per day — normal use won't come close.
