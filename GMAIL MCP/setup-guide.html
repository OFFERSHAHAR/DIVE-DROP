בדיקת דרישות מקדימות

וודא שיש לך את הדברים הבאים לפני שמתחילים:

  • Python 3.10 ומעלה מותקן — הריצו python --version לבדיקה
  • חשבון גוגל (Gmail)
  • כל הקבצים בתיקיית opencode/
מוכן: כל הספריות כבר מותקנות. הריצו pip install -r requirements.txt אם תעברו למחשב אחר.
1

כנסו ל־Google Cloud Console

פתחו את המסוף של גוגל קלאוד כדי ליצור פרויקט ולהפעיל את Gmail API.

https://console.cloud.google.com/

התחברו עם חשבון הגוגל שאת המיילים שלו אתם רוצים לקרוא.

2

יצירת פרויקט חדש

בחלק העליון של הדף, לחצו על תפריט הפרויקטים (יכול להראות "Select a project" או שם של פרויקט קיים).

[תפריט בחירת פרויקט — בשורה העליונה של Google Cloud Console]
  • לחצו על "New Project"
  • קראו לו "Gmail MCP" (או כל שם שבא לכם)
  • לחצו על "Create"
  • חכו כמה שניות, ואז בחרו את הפרויקט החדש מהתפריט
3

הפעלת Gmail API

עברו לספריית ה־API והפעילו את Gmail API לפרויקט הזה.

// בתפריט הצדדי: APIs & Services → Library
  • חפשו "Gmail API"
  • לחצו על הכרטיס "Gmail API"
  • לחצו על הכפתור הכחול "Enable"
טיפ: חכו להודעת "API enabled" לפני שממשיכים.
4

הגדרת מסך ההסכמה (OAuth consent screen)

לפני יצירת תעודות, צריך להגדיר את מסך ההסכמה שמשתמשים יראו בעת האימות.

// בתפריט הצדדי: APIs & Services → OAuth consent screen
  • User Type: בחרו "External" (גם אם אתם המשתמש היחיד)
  • לחצו "Create"
  • App name: "Gmail MCP"
  • User support email: כתובת המייל שלכם
  • Developer contact info: כתובת המייל שלכם
  • לחצו "Save and Continue"
  • Scopes: דלגו (לחצו "Save and Continue")
  • Test users: לחצו "Add Users", הוסיפו את כתובת המייל שלכם, ואז "Save and Continue"
  • עיינו בפרטים ולחצו "Back to Dashboard"
חשוב: חובה להוסיף את המייל שלכם כמשתמש בדיקה, אחרת האימות יכשל עם "Access blocked".
5

יצירת תעודות OAuth

עכשיו ניצור תעודות מסוג "Desktop app" שיאפשרו לסקריפט Python שלכם לגשת ל־Gmail.

// בתפריט הצדדי: APIs & Services → Credentials
  • לחצו "Create Credentials" בחלק העליון
  • בחרו "OAuth client ID"
  • Application type: "Desktop app"
  • Name: "Gmail MCP"
  • לחצו "Create"

יופיע חלון עם ה־Client ID וה־Client Secret החדשים שלכם.

[חלון OAuth client created עם כפתור Download]
  • לחצו על "Download JSON"
  • לחצו "OK" לסגירת החלון
6

הכניסו את credentials.json לתיקייה

קחו את קובץ ה־JSON שהורדתם ושימו אותו במקום הנכון.

  1. מצאו את הקובץ (סביר להניח שקוראים לו client_secret_....json)
  2. שנו את השם ל־credentials.json
  3. העבירו אותו לתיקיית opencode/ שעל שולחן העבודה
// התיקייה שלכם צריכה להיראות כך: Desktop/opencode/ ├── credentials.json <-- זה הוספתם עכשיו ├── gmail_mcp.py ├── oauth_setup.py ├── login.html ├── setup-guide.html ├── setup_guide.txt └── requirements.txt
7

הרצת הגדרת OAuth

פתחו טרמינל בתיקיית opencode/ והריצו את סקריפט ההתקנה.

C:\Users\GamingPC\Desktop\opencode> python oauth_setup.py

ייפתח חלון דפדפן שיבקש מכם להתחבר לחשבון הגוגל שלכם.

[מסך התחברות של גוגל → "Choose an account"]
  • בחרו את חשבון הגוגל שלכם
  • לחצו "Advanced" (האפליקציה לא מאומתת — זה צפוי לפרויקטים אישיים)
  • לחצו "Go to Gmail MCP (unsafe)"
  • לחצו "Continue" כדי להעניק גישת קריאה בלבד
הצלחה! תראו "Authentication successful!" בטרמינל. נוצרה תיקיית .gmail_token/ עם הטוקן השמור.
8

חיבור שרת MCP

הוסיפו את שרת Gmail MCP לקובץ התצורה של לקוח MCP (למשל opencode.json או claude_desktop_config.json).

{ "mcpServers": { "gmail": { "command": "python", "args": [ "C:\\Users\\GamingPC\\Desktop\\opencode\\gmail_mcp.py" ] } } }

הפעילו מחדש את לקוח MCP. תראו את כלי Gmail זמינים:

  • gmail_list_messages — רשימת מיילים
  • gmail_get_message — הצגת מייל מלא
  • gmail_search_messages — חיפוש מיילים
  • gmail_list_labels — רשימת תוויות
  • gmail_get_unread_count — כמות שלא נקראו
  • gmail_list_attachments — רשימת קבצים מצורפים
  • gmail_get_profile — פרטי החשבון

למה לא יחסמו אתכם

  • הרשאת קריאה בלבד: אנחנו מבקשים רק gmail.readonly. אין שליחת, מחיקת או שינוי מיילים.
  • API רשמי של גוגל: משתמשים ב־google-api-python-client — אותה ערכת כלים שגוגל מתחזקת. בלי גירוד או IMAP.
  • המתנה חכמה: בשגיאת קצב (429) מחכים יותר עם כל ניסיון + פיזור אקראי.
  • OAuth 2.0: אין שמירת סיסמאות. הטוקן מתחדש אוטומטית. נשמר בתיקייה .gmail_token/ במחשב שלכם.
  • מקומי בלבד: הכל רץ על המחשב שלכם. בלי שרתים של צד שלישי.
  • מכסה נדיבה: Gmail API מאפשר מיליארד שאילתות ביום — שימוש רגיל לא מתקרב לזה.
