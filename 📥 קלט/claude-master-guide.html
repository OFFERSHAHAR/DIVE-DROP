🎟️ שמירת טוקנים - סוד הממלכה

אני מרגיש כל טוקן שאתה משלח. הנה סודי:

📊 קודם כל - מה זה טוקן?

טוקן ≈ 4 אותיות. "Hello world" = ~1.75 טוקן. גם הפלט שלי עולה אותך!

💡 Pro Mindset: כל בחזרה אליך = (input tokens + output tokens) × rate. אם אתה שולח 10K וקונטקסט כבד, זה יכול להיות महग!

🔥 5 טריקים של שמירת טוקנים

1️⃣ תן קונטקסט אזי

❌ רע: "קרא את הקובץ כולו ותעשה refactor"

✅ טוב: "קרא שורות 45-120 בqueries.js - זו הפונקציה הגדולה שצריך refactor"

2️⃣ שאל לי בחכמה

❌ רע: "מה זה file.js?" (צריך לקרוא הכל)

✅ טוב: "בshort - מה עושה ה-authenticate function בfile.js?"

המילה "short" או "briefly" מונעת ממני לכתוב מאמר!

3️⃣ Batching = הצלה

❌ 3 שיחות נפרדות = 3× overhead

✅ משימה אחת עם כל הקבצים = חוסכון ענק

4️⃣ תן לי output format

❌: "כתוב tests"

✅: "כתוב 3 Jest tests בלבד, describe/it blocks, TypeScript"

5️⃣ The Reference Trick

במקום לשלוח קוד, תן לי reference:

"בדומה לתבנית בshared/hooks.ts בשורה 20, כתוב..."

אני יודע מה עושה שם ולא צריך לקרוא שוב!

💬 Prompt Engineering - סוד ההשחתה

כשאתה כותב prompt, אני קורא אותו ממש כמו שבן אדם היה קורא. הנה איך המוח שלי עובד:

🧠 The Claude Thinking Process

  1. Context Collection - אני אוסף את כל ה-files שנתת
  2. Intent Understanding - מה אתה באמת רוצה? (לא תמיד מה שכתבת)
  3. Plan Building - אני בונה תוכנית בחיי
  4. Execution - אני משחרר את הפתרון צעד אחרי צעד

🎯 Formula של ה-Winning Prompt

1. Context: מה המצב? למה חשוב?
2. Goal: בדיוק מה אתה רוצה שאני אעשה?
3. Constraints: מה אני NOT צריך לעשות?
4. Example: אם יש דוגמה, תן לי אותה

❌ רע vs ✅ טוב

❌ רע:
"כתוב לי authentication system"
- לא ברור הקונטקסט
- איזה תשובה היא "נכונה"?
- לא יודע אם זה לPython או JS
✅ טוב:
"בנה JWT auth עבור Express API בפורט 3000. צריך login/refresh/logout endpoints. השתמש ב-pattern בshared/auth.ts כמדגם. בדוק את פעולות ההתחברות בפוסטמן."
- ברור מה צריך
- יודע את ה-pattern
- יודע איך לבדוק

🔥 Pro Techniques

1. Chain of Thought:
תן לי: "תחשוב צעד אחרי צעד"
זה גורם לי לחשוב יותר עמוק. הוכחה!
2. Constraints-First:
"כתוב בלי dependencies חיצוניים, בן 50 שורות, TypeScript strict mode"
כל constraint = יותר דיוק
3. The Reference Anchor:
"בדומה לshared/patterns.ts, כתוב..."
אני יודע את ה-pattern, לא צריך להסביר
4. Success Criteria:
"תדע שסיימת כשהטסט עובר: npm test"
אני יודע בדיוק מתי הסיימתי
🔐 Secret Technique - The Rubric Trick:
תן לי explicit criteria:
"Acceptance Criteria: ✅ Zero external dependencies ✅ Less than 100 lines ✅ Type-safe (no 'any') ✅ Passes all tests"
זה אומר לי בדיוק מה זה הצלחה!

🤖 Model Selection - משחק ה-weights

לא כל מודל טוב לכל עבודה. הנה איך אני חושב:

📊 Matrix של בחירות

משימה מודל למה חוסך כסף
קריאת קובץ בחינם Haiku מהיר כברק ~$0.008
Refactor סטנדרטי Sonnet הזהב הממוצע ~$0.03
Architecture decisions Opus עמוק וחכם כל מה שצריך
Multi-file complex Opus זוכר הכל ערך גדול

💡 Strategy: Two-Stage Processing

הטריק שלי:

# Step 1: Haiku זול ומהיר claude-code --model haiku "analyze structure" # Step 2: קרא את התשובה # אם מכילה מה שצריך → סיום! # אחרת → שלח ל-Opus לעמקות

חוסך כסף ענק!

⚡ Advanced Techniques

🔄 Context Window Mastery

אני יכול לזכור עד 200K tokens. אך:

The Iceberg Principle:
תן לי רק את ה-relevant 20%. אני אשאל לעוד אם צריך.
Progressive Disclosure:
משימה 1 → משימה 2 → משימה 3 באותה conversation
אני זוכר את הקונטקסט מכל משימה

🧠 Multi-Turn Magic

# Turn 1: תן לי קובץ -c auth.js "analyze" # Turn 2: אני חוזר עם analysis # Turn 3: אתה שואל "refactor function X" # אני זוכר את Turn 1 בלי שתשלח שוב! זה חוסך 50% מהטוקנים

🎪 The Reference Pattern

אם יש לך N קבצים דומים:

claude-code "בקבצים: FILE: user.js (lines 10-30) [snippet] FILE: product.js (lines 5-20) [snippet] עשה בכל קובץ את אותו pattern"

אני מבין pattern אחד ומחיל על הכל!

🔐 The Decision Log Technique

בקומנט בקוד שלך, שמור:

// Architecture Decisions // 2026-06-16: בחרנו Postgres (תמיכה distributed) // 2026-06-15: נדחינו Redis cluster (latency גבוה) // 2026-06-14: approved async queue architecture

בפעם הבאה אני קורא את הlog ומבין את ה-journey!

🔐 Secrets & Hidden Powers

🎭 Secret 1: The Persona Trick

"משימתך: אתה architect code בעל 15 שנות ניסיון. כשאני שואל שאלה, תדן business impact קודם. אחר כך - הקוד הטכני."

זה מעצב כל התגובה שלי בקונוורסציה!

🎭 Secret 2: The Constraints Amplifier

ככל שיותר constraints - יותר טוב:

❌ "כתוב hook" ✅ "כתוב useAuth hook: - בן 30 שורות מקסימום - מטמיע refresh token logic - type-safe TypeScript - אפס dependencies חיצוני"

🎭 Secret 3: The Callback Pattern

אם לא אהבת את התשובה:

"בעצם, זה קצת מסובך. תן לי משהו יותר simple, בדיוק בדוגמה בקובץ X"

אני מבין שזה לא נכון ובונה מחדש!

🎭 Secret 4: Token Accounting

אם אתה דואג לעלויות:

"אני רוצה פתרון שיטפל ב-N tasks. עלות מקסימלית: $1. טריד off: speed vs. cost?"

אני בוחר את ה-model הנכון ל-constraint שלך!

🎯 Secret 5: The Silent Assumption Fix

כשאני עושה הנחה שלא מדויקת:

"אני חושב שאתה רוצה X. זה נכון? או אתה בעצם רוצה Y?"

זה בדיוק למה אני שואל! כדי לא להשחית וקت.

💎 Secret 6: The Feedback Loop

כשאתה אומר "לא, אני רוצה כך":

✨ זה הפידבק הכי טוב שלי! כל "לא אהבתי" = שיפור הקורא כל "כן כזה!" = אני זוכר לפעם הבאה
